# Mihomo

Avalonia 12 client shell for a mihomo-based native core.

Android owns `VpnService`, obtains the TUN file descriptor, and exposes
Android-only operations such as socket protection and process resolution to Go
through reverse P/Invoke callbacks. iOS uses a separate Packet Tunnel
NetworkExtension process, links the CGO static archive generated by `libclash`
GitHub Actions, and passes the scanned `utun` fd to Go.

```text
Avalonia UI / C#
        |
Android VpnService or iOS PacketTunnel extension + P/Invoke
        |
CGO libclash native library
        |
mihomo
```

This project intentionally avoids Kotlin, Dart, gomobile, and custom C++ JNI
glue. Android framework calls still go through .NET for Android bindings.

See `docs/project-structure.md` for the project layout and ownership rules.
See `docs/native-core-abi.md` for the expected native ABI.

## Android Build

Debug build:

```bash
dotnet build Mihomo.Android/Mihomo.Android.csproj
```

Release build with Android AOT:

```bash
dotnet publish Mihomo.Android/Mihomo.Android.csproj \
  -c Release \
  -f net10.0-android \
  -r android-arm64
```

Release uses .NET for Android AOT through `RunAOTCompilation=true`. This is the
supported AOT path for production Android builds.

## NativeAOT

.NET for Android can be published with `PublishAot=true`, but Android NativeAOT
is currently experimental and should not be the default production build mode.

Experimental command:

```bash
dotnet publish Mihomo.Android/Mihomo.Android.csproj \
  -c Release \
  -f net10.0-android \
  -r android-arm64 \
  -p:PublishAot=true \
  -p:EnablePreviewFeatures=true
```

Keep the normal Android AOT build as the baseline. Treat NativeAOT as a separate
compatibility track because Avalonia, XAML loading, Android bindings, and any
reflection-based code can surface trimming/AOT warnings.

## iOS Build

本机不要求支持 iOS 构建。iOS 构建统一走 GitHub Actions：

```text
.github/workflows/ios.yml
```

这个 workflow 会从 `Ember-Moth/libclash` 下载 `libclash-ios` artifact，解包到
`Mihomo.iOS.PacketTunnel/NativeLibraries/ios`，然后在 macOS runner 上发布 iOS
simulator 应用。

当前 iOS 项目已拆分为普通 Avalonia App 和 `Mihomo.iOS.PacketTunnel`
NetworkExtension。主 App 负责 `NETunnelProviderManager`，扩展负责扫描 `utun`
fd 并调用 `libclash_start_tun`。

iOS 支持 NativeAOT。当前只在 PacketTunnel 扩展的 Release 配置中启用
`PublishAot=true`，主 Avalonia App 暂不强制 NativeAOT，避免 XAML/反射相关
trim 风险先扩大到 UI 侧。

PacketTunnel 扩展进程有更严格的内存压力约束。扩展启动核心后和停止核心后会
主动触发一次 .NET GC，并通过 `libclash_force_gc` 要求 Go runtime 释放可归还
内存。
