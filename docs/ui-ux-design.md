# Aureline UI/UX 方案

本文定义 Aureline 自有 UI 语言和后续界面改造方向。目标是摆脱对 FlClash
界面结构、组件外观和动效的复刻，把 Aureline 做成一个清晰、克制、可长期维护的
移动端代理客户端。

## 设计目标

- 建立 Aureline 自有视觉语言，围绕“曜线”这个名称表达网络路径、流量信号和连接状态。
- 优先服务高频操作：启动 VPN、查看当前网络状态、切换出站模式、切换策略节点、导入或更新订阅。
- 保持移动端效率：页面信息密度适中，控件触达面积稳定，横竖屏和不同 DPI 下不挤压、不重叠。
- 兼容多 core 变体：Go/mihomo、clash-rs、meow 的能力差异必须可被 UI 表达，不把缺失能力伪装成可用功能。
- 同时覆盖 Android 和 iOS：共享 UI 保持一致，平台差异只体现在 VPN 权限、iOS PacketTunnel IPC、系统设置入口等局部流程。

## 非目标

- 不复刻 FlClash 的页面布局、卡片样式、图标、动画或配色。
- 不做营销式首页。启动后的第一屏就是可操作的运行控制台。
- 不为了装饰堆叠卡片、渐变球、插画背景或大面积品牌文案。
- 不把无法调用的能力做成静态 UI。所有按钮和选项要么能执行，要么明确处于禁用状态并给出原因。

## 产品定位

Aureline 是一个“网络运行台”，不是纯设置工具。用户打开 App 时最关心三件事：

1. 当前是否接管网络。
2. 流量正在走哪里。
3. 如果不符合预期，怎样快速切换策略、配置或运行模式。

界面应当像轻量运维面板：稳定、可扫读、状态明确。视觉可以有品牌性，但不能牺牲判断速度。

## UI 语言：曜线

“曜线”可以被抽象成三类视觉元素：

- 线：网络路径、策略组、节点链路、流量走势。
- 光：运行状态、连接质量、延迟反馈、重要操作。
- 面：承载信息的安静背景，不抢夺状态和操作的优先级。

### 视觉原则

- 低装饰、高信号：视觉重心放在状态、曲线、选择结果、异常提示。
- 分层清晰：页面背景、内容区域、可交互行、危险操作使用不同层级，而不是全部做成同一种卡片。
- 动效表达方向：页面切换、策略组切换、节点测速应表达“流动”和“路径变化”，不做无意义弹跳。
- 数字优先：流量、延迟、连接数、更新时间使用一致的数字排版和单位规则。
- 平台友好：Android 和 iOS 使用同一套共享样式，但允许系统权限弹窗、状态栏、安全区域按平台处理。

### 颜色系统

建议使用语义 token，而不是在页面里写死颜色。

```text
Background        页面底色
Surface           普通内容面
SurfaceRaised     浮起面，用于底部弹层或临时面板
Stroke            分隔线和边框
TextPrimary       主文本
TextSecondary     次文本
TextMuted         辅助说明
Accent            品牌主色，用于主操作和当前选择
Route             策略、路径、节点相关强调
Success           可用、已连接、低延迟
Warning           网络弱、需要确认
Danger            停止、删除、失败
ChartUpload       上传曲线
ChartDownload     下载曲线
```

推荐色彩方向：

- 浅色：接近白色的冷灰背景，暖金 Accent，青蓝 Route，绿色 Success，珊瑚 Danger。
- 深色：近黑石墨背景，降低饱和度的金色 Accent，清晰但不刺眼的青蓝和绿色。

不要让 UI 被单一蓝紫、奶油色或深蓝色完全支配。Accent 只用于当前选择、主按钮和关键状态，不能成为所有组件的默认填充色。

### 字体和排版

- 页面标题保持短，不使用大 hero 字号。
- 工具页、列表页和状态行使用紧凑标题，避免在小面板里出现过大的显示字。
- 数字使用等宽或接近等宽的显示风格，确保网速、延迟、连接数刷新时不抖动。
- 不使用负字距，不按 viewport 宽度缩放字体。
- 中文、英文、数字混排时，单位用短格式：`12.4 MB/s`、`86 ms`、`1.2 GB`。

### 圆角和边框

- 基础圆角不超过 8px。
- 列表行和按钮优先使用轻边框、底部分隔线或背景层级。
- 避免卡片套卡片。需要分组时使用 full-width section 或列表区块。

### 图标

- App 自有图标走 `Aureline/Assets/AurelineLogo.svg` 和平台图标资源。
- 功能图标优先用当前项目可用的图标字体或后续统一图标库，保持线宽一致。
- 图标按钮必须有 tooltip 或可访问名称。
- 不继续使用 FlClash 的图标语言作为标准。已有临时资源可以在迁移期保留，但新页面不依赖它。

## 信息架构

保留四个一级入口，但重新定义命名和职责：

```text
总览      当前连接、实时流量、网络检测、运行事件
策略      出站模式、策略组、节点选择、测速和延迟
配置      订阅、配置文件、远程更新、当前启用配置
工具      应用设置、核心设置、数据维护、诊断
```

底部导航规则：

- `总览`、`配置`、`工具`始终显示。
- `策略`在有可用运行时数据时显示；如果未启动 VPN，但已有配置，可以显示入口并呈现“启动后可切换”的空状态。
- 底部导航图标和文字必须在各自 item 内居中，不随页面内容宽度偏移。

顶部区域规则：

- 顶部 app bar 不承担大面积状态展示，只放品牌、当前 profile 简写、设置或更多操作。
- 启动/停止 VPN 是全局主操作，固定在总览页核心区域；其他页可通过顶部状态条显示当前运行状态，但不重复放大型开关。

## 页面方案

### 总览

总览页是运行控制台，优先回答“现在是否正常”。

内容顺序：

1. 运行状态区。
2. 实时流量图。
3. 当前网络检测。
4. 内网地址和连接统计。
5. 最近事件。

运行状态区：

- 展示当前状态：未启动、启动中、运行中、停止中、错误。
- 展示当前配置名、出站模式、当前默认策略选择。
- 主按钮根据状态变换：启动、停止、重试、授权 VPN。
- Android 首次启动需要 VPN 权限时，按钮直接触发权限流程。
- iOS 首次启动需要保存并启动 PacketTunnel 时，按钮进入系统授权流程。

实时流量图：

- 曲线图是总览页主要可视元素。
- 上传和下载两条曲线分色显示，最近 60 秒为默认窗口。
- 当前速率显示在曲线右上或图下方固定区域，数字刷新不改变布局。
- 无数据时显示平静空状态，不显示全 0 的假仪表盘。

网络检测：

- 展示当前出口 IP、地区、运营商或 ASN。
- 检测失败时显示失败原因和重试按钮。
- 不把 timeout 写死成普通文本。失败状态必须区别于未检测状态。
- 检测请求应走当前代理路径，必要时标注“直连检测”和“代理检测”的结果差异。

内网地址：

- Android 显示 VpnService 分配的 TUN 地址。
- iOS 显示 PacketTunnel network settings 中配置的 tunnel 地址。
- 如果 core 或平台无法返回，显示“未分配”，不要显示假数据。

最近事件：

- 记录启动、停止、配置加载失败、订阅更新、节点切换、测速失败等事件。
- 只展示最近 5 条，更多日志进入工具页诊断。

### 策略

策略页用于切换出站模式、策略组和节点。

页面结构：

```text
出站模式 segmented control
策略组横向索引
当前策略组摘要
节点列表
底部批量操作或测速进度
```

出站模式：

- 支持 Rule、Global、Direct、Script 等 core 实际支持的模式。
- 当前模式来自 runtime 查询，不写死。
- 切换后立即调用 runtime，失败时回滚 UI 状态。

策略组：

- 横向切换使用项目自有动效：线性滑动和底部光线指示，不复制 FlClash 动画。
- 策略组 item 只显示名称、类型、当前节点、健康状态。
- 组数量多时使用可滚动 pill row，但 item 宽度稳定。

节点列表：

- 节点行整行可点，不能只有 content 区可点。
- 使用列表行或紧凑 tile，不做大面积卡片堆叠。
- 每个节点显示：名称、类型、延迟、上次测试时间、当前选择标记。
- 可选显示地区或协议标签，但不挤压节点名。
- 延迟颜色按语义分级：低延迟 Success，中等 Warning，失败 Danger，未测 Muted。

测速：

- 单节点点击延迟值或测速图标触发该节点测速。
- 策略组支持“测试当前组”。
- 全局测速放入更多菜单，避免误触发大量请求。
- 测速中显示 per-node 进度，不锁死整个页面。

### 配置

配置页只负责 profile 和订阅，不放运行参数。

页面结构：

```text
当前启用配置
配置列表
添加入口
更新状态
```

当前启用配置：

- 展示名称、来源、本地路径、更新时间、是否有未应用更新。
- 提供启用、更新、校验、复制路径、删除。

配置列表：

- 本地配置和远程订阅统一成 profile item。
- item 展示来源类型：本地文件、URL、扫码、备份恢复。
- 点击 item 进入详情，不直接弹出难以维护的复杂菜单。

添加入口：

- URL 导入：默认 UA 使用 `clash`，可在高级选项里调整。
- 本地文件导入：复制到应用 profile 目录并校验。
- 二维码导入：Android 可接相机或图片解析；iOS 后续接系统选择器。
- 剪贴板导入：只在检测到 URL 或 YAML 特征时提示。

配置详情：

- 显示 profile 元数据、订阅地址、上次更新时间、文件大小、校验结果。
- 远程订阅可设置自动更新间隔。
- 删除是危险操作，使用确认底部弹层。

### 工具

工具页是系统能力集合，但必须避免“所有东西都只是能点进空页面”。

分组建议：

```text
应用
  主题
  语言
  启动行为
  通知

核心
  TUN 设置
  DNS 设置
  Geo 低内存模式
  外部控制
  内存控制

数据
  备份与恢复
  清理缓存
  清理连接
  重置应用状态

诊断
  日志
  运行时信息
  Native core 信息
  网络自检
```

工具子页面规则：

- 子页面不是通用“保存/校验/重载”模板。
- 能即时生效的设置使用 switch、segmented control 或 picker。
- 需要保存的复杂表单才显示保存按钮。
- 危险操作放在页面底部危险区。
- 返回键返回上一级工具页，不直接退出 App。

Geo 低内存模式：

- 作为 core 设置项，只在对应 core 支持时可用。
- 开启后说明影响：减少 Geo 数据驻留或切换低内存加载策略，可能影响首次匹配速度。
- 如果当前 core 只支持占位 API，UI 要显示“不支持当前核心”，不能假装生效。

内存控制：

- Android 默认只显示当前 native core 类型和内存估算。
- iOS PacketTunnel 可显示扩展内存预算、最近 trim 时间和手动 trim 按钮。
- 不建议暴露“每秒 GC”作为默认开关。应优先按事件触发和内存阈值触发。

## 状态模型

共享 ViewModel 应显式建模这些状态：

```text
RuntimeState
  Stopped
  PermissionRequired
  Starting
  Running
  Stopping
  Error

ProfileState
  Empty
  Selected
  Validating
  Updating
  Invalid

ProxyState
  Unavailable
  Loading
  Ready
  Switching
  Error

NetworkCheckState
  Idle
  Running
  Success
  Failed
```

UI 只根据状态渲染，不根据零值猜测。比如流量为 0 和未连接不是同一种状态。

## 动效策略

动效服务于方向和状态，不服务于装饰。

- 页面切换：轻量横向滑动，距离短，时长 180ms 左右。
- 策略组切换：内容区跟随滑动，顶部组索引使用线形指示器。
- 启动 VPN：主按钮显示进度环，状态区用短脉冲表示正在建立 tunnel。
- 测速：节点延迟位置显示小型进度，不让整行闪烁。
- 错误：使用静态错误区和可操作按钮，不做强烈抖动。

## 可访问性和输入

- 所有图标按钮要有 `AutomationProperties.Name`。
- 触摸目标不小于 44dp。
- 深色和浅色模式都要满足文本对比度。
- 支持系统返回键：优先关闭弹层，其次返回子页面，再返回一级页，最后才退出。
- 列表 item 整行可点，右侧小按钮只承担附加操作。
- 支持动态文本时，最长节点名、订阅名和错误文本不能挤出容器。

## 多 core 能力表达

Aureline 同时存在多个 Android 变体。UI 不能默认假设所有 core 都支持同一能力。

建议新增能力模型：

```text
CoreCapabilities
  SupportsProxyGroups
  SupportsProxyDelayTest
  SupportsHealthCheck
  SupportsModeSwitch
  SupportsConnections
  SupportsTrafficStats
  SupportsDnsUpdate
  SupportsMemoryTrim
  SupportsGeoLowMemory
  SupportsExternalController
```

共享 UI 根据 capability 渲染：

- 支持：正常显示。
- 暂不支持：禁用并给出短原因。
- 未知：隐藏高级操作，保留基本启动和配置能力。

这样 Go/mihomo、clash-rs、meow 可以逐步补齐能力，而不会产生“UI 看起来能用但实际无效”的问题。

## 实施路线

### 阶段 1：设计系统清理

- 新建统一 theme token，替换页面里散落的颜色。
- 整理按钮、列表行、section、segmented control、状态徽标、图标按钮样式。
- 删除对 FlClash 视觉资源的新增依赖，新页面只使用 Aureline 自有资源。
- 确认浅色、深色主题切换由共享状态驱动，并和系统主题同步。

### 阶段 2：应用壳和导航

- 重做 `MainView`：顶部状态条、内容容器、底部导航居中和安全区域。
- 统一返回栈：工具子页面、配置详情、弹层都进入同一导航模型。
- 给一级页面定义空状态、加载状态、错误状态。

### 阶段 3：总览页

- 用真实 runtime state 驱动启动按钮。
- 实现 60 秒实时流量曲线。
- 网络检测区接入出口 IP 检测和失败重试。
- 内网地址来自平台 runtime，不再写死。

### 阶段 4：策略页

- 出站模式使用真实查询和切换。
- 策略组切换改成自有横向动效。
- 节点 item 改成整行可点。
- 测速和健康检查按 capability 暴露。

### 阶段 5：配置页

- 配置页只保留 profile 管理。
- URL、本地文件、二维码、剪贴板导入统一进入 profile 创建流程。
- Profile 详情页实现启用、更新、校验、删除。

### 阶段 6：工具页

- 拆分工具子页面，不再使用通用占位模板。
- 实现主题、语言、备份恢复、应用设置、核心设置、诊断日志。
- 对不支持的 core 能力显示禁用原因。

### 阶段 7：iOS 对齐

- iOS 主 App 使用相同共享 UI。
- iOS runtime 能力通过 PacketTunnel IPC 填充 capability。
- 对 iOS 不支持或需要系统权限的入口，使用平台化说明和跳转。

## 验收标准

- 页面中不再出现 FlClash 风格的一比一布局、图标或动画复刻。
- 所有一级页面都有真实业务状态，不出现“全是 0 但看似正常”的假数据。
- 启动、停止、切换模式、切换节点、更新订阅、测速这些高频操作都能执行或明确禁用。
- 浅色和深色模式都完整可用。
- Android 返回键行为符合用户预期。
- Android 三个 core 变体都能通过 capability 显示差异。
- iOS 主 App 不直接调用 native core，继续通过 PacketTunnel IPC 对齐共享 UI。

## 迁移原则

- 先替换结构，再替换细节。不要在旧页面上继续堆更多局部修补。
- 每次改一个页面时，同时补齐该页面的空、加载、错误、运行中状态。
- 新控件先进入 `Styles/Controls.axaml` 或独立 shared control，再在页面使用。
- 不新增静态占位功能。暂时无法实现的功能应进入 capability 或 TODO 文档，而不是做成能点但无效的按钮。
- 每个阶段完成后在真机检查：触摸区域、返回键、深浅色、横向滑动、长文本、启动停止流程。
